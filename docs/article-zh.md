# 我让 AI 帮我实现了《原子习惯》里最难做到的那件事

《原子习惯》我读了三遍。

每次读完都觉得「对，就该这样」。然后两周后又回到原来的样子。

James Clear 在书里说：**追踪你的习惯是改变行为的关键。** 他有一张表格，让你每天打勾。

我坚持了 4 天。

不是因为懒，是因为手动记录这件事本身就是一个习惯——而你在还没建立新习惯的时候，去建立「记录习惯的习惯」，本质上是在堆叠两件困难的事。

---

## 我做了一个工具，让 AI 代替我追踪

不需要我做任何事。它在后台观察我的数字行为：
- 我在哪些 repo 上做了多少 commit
- 我一天之间在几个项目之间切换
- 哪些任务开了头但没有结尾
- 我的高产模式和低产模式有什么区别

每周它自动跑一次，Claude 把这些数据合成一份报告。没有 dashboard，没有 app，没有账号。就是纯文本，放在我的电脑里。

---

## 这是上周的真实报告片段

**不是示例数据。是我自己本周的情况。**

```
专注评分：6 / 10

本周 329 次 commit，分布在 12 个 repo。
高产出，但持续性碎片化——你从未真正离开过任何一个代码库。

发现的模式：
4月13日产出了 202 个 commit，是本周日均的 9.2 倍。
前四天（4月7-11日）日均 8 个 commit。
4月12日有什么东西被触发了（68个），延续到了13日。
这不是随机波动，这是一次模式切换。
当你进入那个状态，产出乘以了9倍。

时间现实：
本周 7 天里，有 5 天的产出不到你实际能力的 10%。
你的上限存在。你的下限才是问题。
```

这段话，我读了三遍。

不是因为数字好看。是因为 **我不知道 4 月 13 日发生了什么。**

那天我做了什么不一样的事？没有会议？专注在某一个 repo？还是别的原因？

我不知道。这才是真正的问题。

---

## 《原子习惯》说的是这个

James Clear 的第三条法则：**Make it Obvious — 让它显而易见。**

你改变不了你不知道的东西。

在看到这份报告之前，我以为自己效率还不错。「今天提交了代码，不错。」

但数据告诉我：你有 9 倍产出的潜力，5 天里你在用不到 10% 的它。

手动追踪永远找不到这个。因为你无法主动追踪你不知道存在的模式。

**自动观察才能。**

---

## 工具告诉我的另外几件事

**12 个 repo 活跃，大部分没有完结**

`news`、`uyux`、`x-agent-waitlist` 这周都动了，但每个都只有少量触碰。开了头，没有结尾。

工具估计：**8-10 个开放的未完成线程**。

我以为自己在推进多个项目。实际上我在不断地开启、不断地搁置。

**工具给的建议不是通用建议**

它不是说「提高专注力」、「减少切换」这种废话。

它说的是：**「先搞清楚 4 月 13 日发生了什么，然后把那个条件复制出来。」**

这是从我的数据里推导出来的，不是从某本书里抄的。

---

## 这个工具叫 Mirror

核心理念：**零手动输入。你正常工作，AI 观察你。每周日晚上，你看到自己的真实面貌。**

不是你以为的自己，是数字行为反映出来的你。

完全本地运行，没有云，没有账号，没有遥测。开源，免费。

**现在可以用：**

```bash
# 安装（macOS，需要 Python 3.10+）
git clone https://github.com/lessthanno/engram-agent.git ~/engram-agent
bash ~/engram-agent/scripts/install.sh

# 立刻跑一次，看看你本周的数据
python3 ~/engram-agent/core/mirror.py
```

如果你有 Claude API key 或者装了 Claude Code，它会用 AI 生成分析。没有的话，会先生成原始数据，等你配置后再分析。

等待列表和最新进展：**github.com/lessthanno/engram-agent**
